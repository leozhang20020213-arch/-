## 二、规则依据，必须按这些口径改

### 1. 气骰与气海

玩家规则书明确：气骰区域统一为七区：

```text
气池 / 气海 / 锁气 / 阴槽 / 阳槽 / 息库 / 临气区
```

规则口径：

- 气池：未投掷总池，不能用于成招或配置。
- 气海：已投掷、当前可调度的气骰池，玩家从这里选骰。
- 锁气：本次宣言中锁定投入的气骰。
- 阴槽/阳槽：本次配置的槽位，计算阴值、阳值、合值、阴阳差。
- 息库：已使用气骰的回收区。
- 临气区：临时气骰区，使用后通常消失。

界面要求：

- 不要同时保留“上方四块气海/锁气/息库/临气摘要”和“下方气海 Dock”两个主操作区。
- 气海必须能直接看到骰子本体、点数、气性、骰阶和可拖拽状态。
- 下方 Dock 应成为唯一的气骰操作主区。
- 上方如果需要显示气骰，只能做极简状态条或完全删除，不得与 Dock 重复。
- 调息、返照属于气骰回收动作，应靠近息库/气海主操作区，不要分散成两个独立气海区。

### 2. 宣言与目标线

DM 规则书明确：目标线用于处理“谁正在影响谁”，基本写法：

```text
[行动方] → [承受方]
```

规则口径：

- 目标线不是单纯装饰线，也不是普通“当前目标”列表。
- 目标线要服务于宣言、截击、应招、护人、承接、转移等窗口。
- 宣言前必须明确目标线，否则后续响应会混乱。

界面要求：

- 战场中心可以显示当前有效目标线，但必须带文字标签，例如“沈青 → 短兵客｜近身｜破浪横刀”。
- 不要在左侧角色卡下方再放“当前目标”敌人列表。该模块与战场敌人卡、右侧目标下拉、目标线重复，应删除。
- 右侧宣言面板保留目标选择，但选择后要同步到战场目标线和底部提示。
- 目标线状态应区分：
  - 未选目标：提示“点击战场敌人或右侧目标下拉选择”。
  - 已选目标未锁气：显示“行动方 → 承受方”。
  - 已锁气：显示“行动方 → 承受方｜招式名｜锁气中”。
  - 被截击改变：显示“目标线已改变/失效”，并进入合法性提示。

### 3. 危机值与解密值

DM 规则书明确：

- 危机值记录场景中正在逼近的麻烦，必须命名，有增长条件和触发落果。
- 解密值记录玩家已经掌握并能够使用的信息，应按信息深度分层。
- 两者可以同时推进，但功能不同。

界面要求：

- 不要把“解密值 0/8”和“危机值 0/8”做成两个无名进度条，这会重复且无信息。
- 应改成“场景轨”模块，按类型展示：
  - 危机：显示危机名、当前值、触发阈值、下一触发落果。
  - 解密：显示当前层级、已公开线索摘要、下一层未公开或 DM 私有提示。
- 玩家视角只显示公开内容；DM 视角可以看到增长条件、触发落果和隐藏层内容。
- 如果 mock 数据暂时没有这些字段，先扩展 mock 类型和数据，不要用两个空进度条占位。

### 4. 轮次、先后手与出手顺序

玩家规则书和 DM 规则书都要求交锋准备时计算先后：

```text
先后值 = 身势或经 DM 许可的观照 + 当前最高可用气骰点数 + 状态修正
```

新手内测可每场交锋开始算一次，进阶桌可每轮重算。

界面要求：

- 顶部栏不能只显示“第1轮 → 沈青 → 情景中”。
- 顶部栏或其下方应显示本轮行动顺序条，至少包含：
  - 顺位编号
  - 角色名
  - 先后值
  - 已行动/当前行动/待行动状态
  - 当前激活角色高亮
- 如果空间不足，顶部显示紧凑顺序条，详细计算放到 hover/title 或 DM 面板。
- 先后手计算要在 mock 数据中体现，不能只写固定排序。
