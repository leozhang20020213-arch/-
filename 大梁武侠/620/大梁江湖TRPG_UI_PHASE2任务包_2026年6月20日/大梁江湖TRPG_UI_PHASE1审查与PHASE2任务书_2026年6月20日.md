# 大梁江湖TRPG_UI_PHASE1审查与PHASE2任务书_2026年6月20日

## 一、PHASE1 审查结论

PHASE1 可以通过，但不得直接进入美术填充。下一步必须先做“规则字段只读绑定 + 交锋主屏可截图验收”。

### 1. 已符合 R6 的部分

1. 已使用项目内便携 Godot 4.7，不做全局安装。
2. 已备份旧 React/Phaser UI，并停用旧 UI 默认启动入口。
3. 已建立 Godot 1920×1080 Windows 主框架。
4. 已建立交锋主屏卡牌操作骨架：
   - 选择卡牌；
   - 选择目标；
   - 宣言/锁气预览；
   - 响应；
   - 结算。
5. 交锋主屏小卡只显示表属性摘要，不显示六根。
6. 调息、返照、取物、争夺物、便行已进入卡牌区。
7. 保留小按钮：
   - 人物；
   - 背包；
   - 装备；
   - 内功；
   - 外功；
   - 法门；
   - 规则；
   - 日志；
   - DM。
8. 六根只在人物、内功、规则抽屉中作为里属性/详情/速查占位显示。
9. 未加入地图、格子、Token坐标、地形、路径、战棋移动。
10. 规则书、世界观、数据库、计算层、规则层、数据层未改动。

### 2. 仍未完成的问题

1. 没有生成截图，无法肉眼验收当前 UI。
2. 第一阶段未接入数据库，所有交锋小卡字段还是占位。
3. 第一阶段未直接解析规则书/数据库原文，字段命名仍未锁死。
4. 美术素材槽位只是文字说明，还没有实际 Theme/NinePatchRect 结构。
5. 没有 rules_ui_manifest.json，后续 Codex 容易继续自由发挥字段。
6. 没有明确卡牌数据结构与规则字段的只读映射。
7. 没有明确抽屉层级、浮动层级、遮罩层级。
8. 没有完成 Phase1 截图存档。

---

## 二、PHASE2 总目标

PHASE2 只做三件事：

```text
1. 建立规则字段只读绑定层。
2. 建立 UI manifest，锁死页面术语、字段和占位。
3. 生成交锋主屏截图，供人工验收。
```

PHASE2 暂时不做完整美术，不挑具体素材，不改规则，不改数据库，不做复杂动画。

```text

│ 交锋页｜第【占位】轮｜准备｜宣言｜计算｜响应｜结算｜势变化｜设置             │
├───────────────┬────────────────────────────────────────────┬─────────────────┤
│ 玩家方          │ 交锋裁决区                                  │ 敌方            │
│ 玩家卡：        │ 当前行动者｜当前目标｜当前卡｜当前阶段        │ 敌人卡：        │

│ 势              │ 目标线 / 纠缠线 / 响应线 / 第三方护人线      │ 势              │
│ 状态            │                                            │ 状态            │
│ 气骰摘要        │ 宣言/锁气预览                              │ 气骰摘要        │
```
