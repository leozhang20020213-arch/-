# 大梁江湖TRPG_UI重构设计书_R6_卡牌操作回滚版_2026年6月20日

## 0. 本版回滚结论

R5 作废。R6 以规则书、世界观、数据库为第一优先级，重新确定 Windows 桌面端 UI。

本版修正：

1. 六根是内属性/里属性，不在交锋主屏小卡显现。
2. 交锋主屏显示表属性摘要，不显示顶门、目窍、心口、丹田、命门、步根。
3. 气骰不是点数点击玩法，不做“逐点点击”“十点点击”。
4. 操作方式改为卡牌游戏逻辑：选择招式卡/操作卡 → 选择目标 → 系统按规则进入宣言/锁气/响应/结算流程。
5. 调息、返照、取物、争夺物、便行等不做底部裸按钮，纳入操作卡/招式卡体系。
6. 背包、装备、内功、外功、法门、规则、日志以小按钮/浮动抽屉方式保留在交锋主界面。
7. 本规则没有地图，不做地图、格子、Token 坐标、地形、路径、战棋移动。
8. 所有术语、页面语言、字段名从规则书、世界观、数据库读取；不能确认的内容只写【占位：字段名】。

## 1. 固定术语

### 1.1 六根/内属性/里属性

六根固定为：

```text
顶门｜目窍｜心口｜丹田｜命门｜步根
```

六根只在以下页面显示：

```text
人物页
内功抽屉
人物详情浮窗
数据库维护页
规则速查页
```

六根不在交锋主屏玩家小卡/敌人小卡常驻显示。

### 1.2 交锋主屏可见表属性摘要

交锋小卡只显示表属性摘要。表属性具体字段以规则书/数据库为准，当前 UI 不擅自创造字段。

交锋小卡固定显示：

```text
【占位：角色名】
势：【占位：势】
状态：【占位：状态】
气骰：【占位：气骰摘要】
距离：【占位：距离】
纠缠：【占位：纠缠】
当前：【占位：当前宣言/响应】
```

如规则书/数据库后续明确表属性名称，替换占位；未明确前不得写六根，不得自造“攻击、防御、速度、血量、蓝量”等外部游戏术语。

### 1.3 气骰五区

固定为：

```text
气池｜气海｜锁气｜息库｜临气区
```

### 1.4 交锋阶段

固定为：

```text
准备｜宣言｜计算｜响应｜结算｜势变化
```

### 1.5 势面

固定为：

```text
阴盛｜阳盛｜合势｜圆融｜崩溃｜失势
```

### 1.6 主操作词

固定为：

```text
宣言｜锁气｜截击｜成招｜应招｜落果｜调息｜返照｜取物｜争夺物｜便行
```

### 3.2 交锋页操作流程

不再做逐点点击。流程固定为：

```text
1. 玩家选择一张卡：
   招式卡 / 调息卡 / 返照卡 / 取物卡 / 争夺物卡 / 便行卡

2. UI 展开选中卡详情：
   阴阳结构、投入、势要求、距离、效阶、落果

3. 玩家选择目标：
   点击玩家卡/敌人卡/自己/无目标占位

4. 系统进入宣言：
   当前卡进入宣言/锁气预览区

5. 系统根据规则与当前气骰五区显示锁气结果：
   不要求玩家点击单个点数
   不要求玩家点击“十点”
   只允许整卡确认、整组换骰、取消宣言、改选目标

6. 若可响应：
   响应窗口浮起，相关角色可选择截击卡/应招卡

7. 结算：
   显示效阶、落果、势变化、气骰去向
```

### 3.3 气骰交互原则

```text
气骰只作为资源状态和锁气结果显示。
玩家不得逐点点击点数。
如需要手动调整，只允许点击“整枚气骰”或“换一组投入方案”。
UI 默认按招式卡的阴阳结构和当前气海推荐投入。
```

## 8. 引擎、模块、教程、素材策略

### 8.1 引擎

使用 Godot。

原因：

```text
免费开源
适合 Windows 桌面端
Control / Container / Theme / NinePatchRect 适合固定 UI
可由 Codex 直接创建场景、脚本、主题、资源目录
```

### 8.2 UI 技术路线

```text
Control：所有界面根节点
Container：固定区域排版
Theme：统一皮肤
NinePatchRect：可缩放卡框、面板、抽屉
AnimationPlayer / Tween：浮窗、抽屉、卡片悬停
Resource / JSON：规则和数据库只读绑定
```

### 8.3 模块优先来源

统一从 Godot Asset Library / GitHub 调研，不直接改规则逻辑。

```text
卡牌拖拽/选择：只借鉴卡牌 UI 操作
背包系统：只借鉴格子和抽屉 UI
Dialogue/提示系统：只借鉴规则提示浮窗
Quest/场景系统：后续可借鉴场景轨，不进入第一阶段
```

### 8.4 教程学习方向

Codex 必须先学习以下方向：

```text
Godot Control / Container / Theme
Godot NinePatchRect / StyleBoxTexture
Godot 卡牌拖拽与卡牌选择
Godot 背包 Inventory UI
Godot 浮窗、抽屉、Tween 动画
```

### 8.5 素材策略

第一阶段不挑具体素材，只留下槽位。

后续素材填充原则：

```text
优先同一网站
优先同一作者
优先同一套 UI 包
优先武侠/东方/卷轴/纸面/木案/墨色/朱砂方向
不混搭多个风格
```

## 11. Codex 总提示词

```text
你现在执行“大梁江湖TRPG_UI重构_R6_卡牌操作回滚版”。

最高优先级：规则书、世界观、数据库原文。

R5 作废。交锋主屏不显示六根。六根是内属性/里属性，只在人物页、内功抽屉、人物详情、数据库维护、规则速查中显示。

交锋小卡只显示表属性摘要：
角色名、势、状态、气骰摘要、距离、纠缠、当前宣言/响应。
表属性具体名称以规则书/数据库为准；不能确认的写【占位：字段名】。

操作方式必须像卡牌游戏：

禁止逐点点击。禁止让玩家点“十点”。气骰只作为资源状态和锁气结果显示；如需调整，只允许整枚气骰或整组投入方案。

调息、返照、取物、争夺物、便行都进入卡牌区，不做底部裸按钮。

主界面必须保留小按钮：
人物、背包、装备、内功、外功、法门、规则、日志、DM。
点击小按钮打开浮动抽屉。

禁止：
地图、格子、Token坐标、地形、路径、战棋移动。
禁止添加规则书、世界观、数据库之外的术语。
不能确认字段，一律写【占位：字段名】。

每阶段必须输出 README_UI_PHASEX.md，说明：
完成内容、修改文件、未改动文件、调用规则字段、调用数据库字段、美术素材槽位、规则冲突、截图路径、下一步建议。
```
