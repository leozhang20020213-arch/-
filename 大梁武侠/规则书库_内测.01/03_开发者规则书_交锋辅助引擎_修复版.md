## 一、App边界

App只辅助交锋、气骰、状态、长效被动、场景轨和日志回放。不处理完整自由叙事、NPC自然语言对话、玩家推理正确性、世界观开放裁定。

## 二、核心区域枚举

气骰七区（核心五区 + 息库 + 临气区）：

| 区域 | 标识 | 用途 |
|---|---|---|
| 气池 | QI_POOL | 未投掷总池，角色长期持有的常规气骰存放处 |
| 气海 | QI_SEA | 已投掷、当前可调度的常规气骰区 |
| 锁气 | QI_LOCK | 本次宣言中已锁定投入的气骰区 |
| 阴槽 | QI_YIN | 本次配置中分配到阴面的气骰区 |
| 阳槽 | QI_YANG | 本次配置中分配到阳面的气骰区 |
| 息库 | QI_REST | 已使用气骰的回收暂存区 |
| 临气区 | QI_TEMP | 临时气骰的存放区，使用后一般消失 |

## 三、气骰生命周期

进入场景时，从气池取常规气骰，投掷后进入气海。一个场景内，气海点数不因情景/交锋切换自动重投。进入另一个新场景时，重新从气池投出。

临时骰在宣言时投，锁气时确定气性。成招并产生效果后进入息库；若被截击成功导致宣言不合法，锁气骰直接进入息库。截击方、应招方与正常招式一样，结算后进入息库。**非调息的常规流转**中，息库气骰回气海不重掷。**调息动作**取回的气骰需重掷后入气海。

## 四、主要流程伪代码

### round_start()
```
round_start():
    for die in character.qi_sea:
        preset_nature(die)               # 按来源预设气性；锁气时以当前来源最终确认
```

### enter_scene()
```
enter_scene(scene_id):
    for die in character.qi_pool.available:
        roll(die)                        # 新场景全部投掷
        move(die, QI_SEA)
```

### declare_action()
```
declare_action(actor, move, target, dice):
    validate_target()                    # 校验目标线合法性
    move(dice, QI_LOCK)                  # 锁气
    set_temp_qi_nature_if_any()          # 临时骰确定气性
    # 常规骰气性在 round_start 时已预设；此处仅重新校验来源是否仍有效
```

### intercept_success_makes_illegal()
```
intercept_success_makes_illegal(action):
    action.cancelled = true
    move(action.locked_dice, QI_REST)
    log()
```

### resolve_move()
```
resolve_move(action):
    check_formed()                       # 成招校验
    calculate_base_effect()              # 基础效果
    calculate_slot_triggers()            # 槽值触发
    open_react_window()                  # 应招窗口
    apply_outcome()                      # 落果
    move(used_dice, QI_REST)             # 已用骰入息库
```

### use_reflection()  [返照]
```
use_reflection(actor):
    require actor.qi_sea.empty           # 断气条件
    consume_action()                     # 消耗行动机会
    die = min(actor.qi_rest.available)   # 取最低阶本命气骰
    move(die, QI_SEA)                    # 放入气海
    no_reroll()                          # 返照不重掷，保持原有点数
```

### regulate_breath()  [调息]
```
regulate_breath(actor):
    dice = select_from_rest(actor)       # 从息库选择符合条件的气骰
    reroll(dice)                         # 调息主动取回并重掷，区别于普通流转
    move(dice, QI_SEA)                   # 放入气海
```

### expire_source()
```
expire_source(source_id):
    remove_unlocked_source_dice()
    if locked_then_revalidate()
    no_rollback_applied_outcomes()
```

## 五、必须保留的手动入口

1. DM手动开始新场景。
2. DM手动确认情景转交锋。
3. DM手动修正气骰来源失效。
4. DM手动写入关系事实、名望事实与自由裁定。
5. DM手动覆盖错误校验，但必须写入日志。

## 六、第一版验收标准

- 能创建角色并录入气骰来源。
- 能进入场景并生成气海。
- 能声明行动、锁气、触发截击、成招、应招、落果。
- 能处理返照、调息、装备/药物来源失效。
- 能记录完整日志并回放每一步气骰去向。
