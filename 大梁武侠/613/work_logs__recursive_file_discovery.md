## 候选文件处理

如果没有找到首选文件名，但发现相似文件名，应按以下规则处理：

1. 记录候选文件路径、文件名、大小、修改时间。
2. 不得自动把候选文件视为等价输入。
3. 对岁月与生成数据库这类 P0 输入，必须在 `missing_files.md` 和状态文件中继续标为缺失，除非文件名或内容能明确证明是同一数据库版本。
4. 若使用候选文件继续做局部审查，结论必须写明“限制性结论”。

## 证据记录格式

后续每条规则抽取、冲突审查、覆盖率审查，都应包含：

```text
source_file: 实际文件绝对路径
source_type: 玩家规则书 / DM规则书 / 数据库 / 审查包 / 报告
chapter_or_sheet: 章节名或sheet名
position: 页码、段落序号、行号或单元格范围
excerpt: 原文短摘录
confidence: 高 / 中 / 低
limitations: 是否受缺失文件影响
```

## PowerShell 查找命令

后续自动循环优先使用：

```powershell
rg --files -g '*.docx' -g '*.xlsx' -g '*.md' -g '*.json' -g '*.jsonl'
```

如需带文件大小和修改时间：

```powershell
Get-ChildItem -LiteralPath 'D:\trpg\大梁武侠' -Recurse -File |
  Where-Object { $_.FullName -notmatch '\\.git\\|__pycache__' } |
  Select-Object FullName, Length, LastWriteTime
```
